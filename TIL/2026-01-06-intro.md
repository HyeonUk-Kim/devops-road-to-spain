# 2026-01-06 실무 리눅스 트러블슈팅 로그

오늘 금융 API 서버 점검 및 업무를 수행하며 활용한 주요 커맨드 정리. 
나중에 DevOps 환경에서 로그 분석 및 리소스 모니터링 시 핵심 무기가 될 것임.

### 1. 로그 분석 및 실시간 모니터링
- `tail -f [logfile] | grep --line-buffered "ERROR"` 
  : 실시간 에러 로그 추적. (금융권 대용량 로그 트래픽에서 특정 키워드 필터링 시 유용)
- `find . -name "*.log" -mtime -1` 
  : 최근 24시간 내 생성/수정된 로그 파일 검색.

### 2. 프로세스 및 네트워크 상태 점검
- `netstat -tulpn` 또는 `ss -tulpn`
  : 현재 개방된 포트와 프로세스 확인. 포트 충돌 해결 시 필수.
- `ps aux --sort=-%mem | head -n 10`
  : 메모리 점유율이 높은 상위 10개 프로세스 확인.
